Mission

Privacy Is the Product Layer

PrivChain integrates with x402 payment semantics and focuses engineering effort on privacy infrastructure.

x402-Compatible

PrivChain aligns with x402 payment flow patterns instead of introducing a competing payment transport.

Privacy Layer

Verification semantics and settlement orchestration are designed for private agent transaction workflows.

MCP-Native

Model Context Protocol tooling enables AI agents to execute payment, settlement lookup, and reconciliation actions.

Architecture

Public System View

Component names and interfaces are public; implementation internals remain private pre-release.

01

Client and Signer

SDK or MCP client prepares a payment intent and obtains signed material via the signer service.

POST /sign-intent
02

Facilitator Verification

Facilitator validates payload shape, replay/idempotency controls, and proof envelope semantics.

POST /verify
03

Settlement and Lifecycle

Settlement is submitted through chain adapters and tracked with lookup + reconciliation interfaces.

POST /settle • GET /settlements/:ref • POST /reconcile
Public Interface Surface

Endpoints and Tools

Tier A-safe interface list. No production hostnames or credentials are published here.

Facilitator Endpoints

  • POST /verify - validate payment payload and privacy envelope
  • POST /settle - submit settlement request through configured adapter
  • GET /settlements/:settlementRef - fetch settlement lifecycle state
  • POST /reconcile - refresh pending settlement statuses in bulk
  • GET /health - service readiness and status

Scheme, Signer, MCP

  • Payment Scheme: priv-stark/v1
  • Signer Endpoint: POST /sign-intent
  • MCP Tools: privchain_pay, privchain_get_settlement, privchain_reconcile
Security Posture

Public Security Summary

High-level controls only. Detailed threat model and infrastructure topology remain private pre-release.

Request Controls

  • Replay protection for payment payloads
  • Idempotency enforcement for verify and settle paths
  • Optional bearer auth for protected deployments

Operational Controls

  • Structured error handling for deterministic clients
  • Settlement lifecycle status and reconciliation path
  • Alerting hooks for stale or failed refresh conditions

Key Management Direction

  • External signer isolation (non-custodial path)
  • AWS KMS backend support
  • HSM-compatible custody path via KMS custom key stores

Developer Availability Intent

SDK and MCP surfaces are being prepared for staged public release once compatibility, security, and disclosure gates are approved.
